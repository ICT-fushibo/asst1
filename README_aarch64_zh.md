### 程序 3，第 1 部分：一些 ISPC 基础知识（共 20 分中的 10 分）###

在阅读 ISPC 代码时，你必须记住，尽管代码看起来很像 C/C++ 代码，但 ISPC 的执行模型与标准 C/C++ 不同。与 C 语言相比，ISPC 程序的多个程序实例总是同时在 CPU 的 SIMD 执行单元上执行。同时执行的程序实例数量由编译器决定（并针对底层机器进行特定选择）。并发实例的数量可以通过内建变量 `programCount` 供 ISPC 程序员使用。ISPC 代码可以通过内建变量 `programIndex` 引用自己的程序实例标识符。因此，从 C 代码调用 ISPC 函数可以被视为生成一组并发的 ISPC 程序实例（在 ISPC 文档中称为一个 gang）。该 gang 的实例运行完成后，控制权返回到调用方的 C 代码。

__停一下。这是你友好的讲师。请再读一遍前一段。相信我。__

例如，以下程序使用常规 C 代码和 ISPC 代码的组合来相加两个 1024 元素的向量。正如我们在课堂上讨论的，由于 gang 中的每个实例都是独立的且执行完全相同的程序逻辑，因此可以通过使用 SIMD 指令的实现来加速执行。

下面给出一个简单的 ISPC 程序。以下 C 代码将调用以下 ISPC 代码：

    ------------------------------------------------------------------------
    C 程序代码：myprogram.cpp
    ------------------------------------------------------------------------
    const int TOTAL_VALUES = 1024;
    float a[TOTAL_VALUES];
    float b[TOTAL_VALUES];
    float c[TOTAL_VALUES]
 
    // 在此处初始化数组 a 和 b。
     
    sum(TOTAL_VALUES, a, b, c);
 
    // 从 sum 返回后，a + b 的结果存储在 c 中。

对应的 ISPC 代码：

    ------------------------------------------------------------------------
    ISPC 代码：myprogram.ispc
    ------------------------------------------------------------------------
    export sum(uniform int N, uniform float* a, uniform float* b, uniform float* c)
    {
      // 假设 programCount 能整除 N。
      for (int i=0; i<N; i+=programCount)
      {
        c[programIndex + i] = a[programIndex + i] + b[programIndex + i];
      }
    }

上面的 ISPC 程序代码在程序实例之间交错处理数组元素。注意这与程序 1 的相似性，在程序 1 中你静态地将图像的部分分配给线程。

然而，与其思考如何在程序实例之间划分工作（即工作如何映射到执行单元），通常更方便且更强大的做法是只关注将问题分解为独立的部分。ISPC 的 `foreach` 结构提供了一种表达问题分解的机制。下面，ISPC 函数 `sum2` 中的 `foreach` 循环定义了一个迭代空间，其中所有迭代都是独立的，因此可以按任意顺序进行。ISPC 处理循环迭代到并发程序实例的分配。下面 `sum` 和 `sum2` 之间的区别很微妙，但非常重要。`sum` 是命令式的：它描述了如何将工作映射到并发实例。下面的示例是声明式的：它只指定了要执行的工作集合。

    -------------------------------------------------------------------------
    ISPC 代码：
    -------------------------------------------------------------------------
    export sum2(uniform int N, uniform float* a, uniform float* b, uniform float* c)
    {
      foreach (i = 0 ... N)
      {
        c[i] = a[i] + b[i];
      }
    }

在继续之前，我们鼓励你通过阅读 <http://ispc.github.io/example.html> 上的 ISPC 入门指南来熟悉 ISPC 语言结构。入门指南中的示例程序与程序 3 中 `mandelbrot.ispc` 对 `mandelbrot_ispc()` 的实现几乎完全相同。在作业代码中，我们更改了 `foreach` 循环的边界，以实现更直接的实现。

## 程序 5：BLAS `saxpy`（10 分）##

程序 5 是 BLAS（基本线性代数子程序）库中 `saxpy` 例程的一个实现，该库在许多系统上被广泛使用（并经过高度优化）。`saxpy` 计算简单的操作 `result = scale*X+Y`，其中 `X`、`Y` 和 `result` 是包含 `N` 个元素的向量（在程序 5 中，`N` = 2000 万），`scale` 是一个标量。注意，`saxpy` 每使用三个元素执行两个数学运算（一次乘法，一次加法）。`saxpy` 是一种*平凡可并行的计算*，具有可预测且规则的数据访问和可预测的执行成本。
