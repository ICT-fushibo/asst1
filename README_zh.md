### 程序 3，第 1 部分：ISPC 基础（10 分，共 20 分）###

在阅读 ISPC 代码时，你必须牢记，尽管代码看起来很像 C/C++ 代码，但 ISPC 的执行模型与标准 C/C++ 不同。与 C 不同，ISPC 程序的多个程序实例总是在 CPU 的 SIMD 执行单元上并行执行。同时执行的程序实例数量由编译器决定（并针对底层机器进行特定选择）。这个并发实例数可以通过内置变量 `programCount` 供 ISPC 程序员使用。ISPC 代码可以通过内置变量 `programIndex` 引用自己的程序实例标识符。因此，从 C 代码调用 ISPC 函数可以被视为启动一组并发的 ISPC 程序实例（在 ISPC 文档中称为一个 gang）。这组实例运行完成后，控制权返回给调用方的 C 代码。

以下示例使用常规 C 代码和 ISPC 代码的组合来将两个 1024 元素向量相加。正如我们在课堂上讨论的，由于 gang 中的每个实例都是独立的并且执行完全相同的程序逻辑，因此可以通过使用 SIMD 指令来实现加速执行。

下面给出了一个简单的 ISPC 程序。以下 C 代码将调用以下 ISPC 代码：

    ------------------------------------------------------------------------
    C 程序代码：myprogram.cpp
    ------------------------------------------------------------------------
    const int TOTAL_VALUES = 1024;
    float a[TOTAL_VALUES];
    float b[TOTAL_VALUES];
    float c[TOTAL_VALUES]
 
    // 在此处初始化数组 a 和 b。
     
    sum(TOTAL_VALUES, a, b, c);
 
    // 从 sum 返回后，a + b 的结果存储在 c 中。

对应的 ISPC 代码：

    ------------------------------------------------------------------------
    ISPC 代码：myprogram.ispc
    ------------------------------------------------------------------------
    export sum(uniform int N, uniform float* a, uniform float* b, uniform float* c)
    {
      // 假设 programCount 能整除 N。
      for (int i=0; i<N; i+=programCount)
      {
        c[programIndex + i] = a[programIndex + i] + b[programIndex + i];
      }
    }

上面的 ISPC 程序代码在程序实例之间交错处理数组元素。注意这与程序 1 的相似性，在程序 1 中你静态地将图像的部分分配给线程。

然而，与其思考如何在程序实例之间划分工作（即工作如何映射到执行单元），更便捷且更强大的方式往往是将重点放在将问题分解为独立的部分上。ISPC 的 `foreach` 结构提供了一种表达问题分解的机制。下面，ISPC 函数 `sum2` 中的 `foreach` 循环定义了一个迭代空间，其中所有迭代都是独立的，因此可以按任何顺序执行。ISPC 处理循环迭代到并发程序实例的分配。下面 `sum` 和 `sum2` 之间的差异很微妙，但非常重要。`sum` 是命令式的：它描述了如何将工作映射到并发实例。下面的例子是声明式的：它只指定了要执行的工作集合。

    -------------------------------------------------------------------------
    ISPC 代码：
    -------------------------------------------------------------------------
    export sum2(uniform int N, uniform float* a, uniform float* b, uniform float* c)
    {
      foreach (i = 0 ... N)
      {
        c[i] = a[i] + b[i];
      }
    }

在继续之前，建议你通过阅读 ISPC 在线教程 <http://ispc.github.io/example.html> 来熟悉 ISPC 语言结构。教程中的示例程序几乎与程序 3 中 `mandelbrot.ispc` 的 `mandelbrot_ispc()` 实现完全相同。在作业代码中，我们更改了 foreach 循环的边界以得到更直接的实现。

程序 4 是一个 ISPC 程序，计算 2000 万个介于 0 和 3 之间的随机数的平方根。它使用了一种快速的迭代平方根实现，采用牛顿法求解方程 ${\frac{1}{x^2}} - S = 0$。该实现中使用值 1.0 作为初始猜测。下图显示了 `sqrt` 在 (0-3) 范围内收敛到精确解所需的迭代次数。（该实现对范围之外的输入不收敛）。请注意，收敛速度取决于初始猜测的准确性。

## 程序 5：BLAS `saxpy`（10 分）##

程序 5 是在许多系统上被广泛使用（并经过高度优化）的 BLAS（基本线性代数子程序）库中 saxpy 例程的一个实现。`saxpy` 计算简单的操作 `result = scale*X+Y`，其中 `X`、`Y` 和 `result` 是由 `N` 个元素组成的向量（在程序 5 中，`N` = 2000 万），`scale` 是一个标量。注意，`saxpy` 每使用三个元素执行两个数学运算（一次乘法，一次加法）。`saxpy` 是一个 *可以平凡并行化的计算*，具有可预测、规则的数据访问和可预测的执行成本。

**你需要完成的任务：**

1. 编译并运行 `saxpy`。程序将报告 ISPC（无任务）和 ISPC（有任务）实现的 saxpy 性能。你观察到使用 ISPC 有任务带来了多少加速比？解释该程序的性能。你认为它可以被大幅改进吗？（例如，你能重写代码以实现接近线性的加速比吗？能还是不能？请证明你的答案。）
2. __额外加分：__（1 分）注意 `main.cpp` 中计算消耗的总内存带宽为 `TOTAL_BYTES = 4 * N * sizeof(float);`。尽管 `saxpy` 从 X 加载一个元素，从 Y 加载一个元素，并向 `result` 写入一个元素，但乘以 4 是正确的。为什么会这样？（提示：思考 CPU 缓存的工作原理。）
3. __额外加分：__（分数根据具体情况处理）提升 `saxpy` 的性能。我们期望看到显著的加速比，而不仅仅是几个百分点的提升。如果成功，请描述你是如何实现的，以及在这些系统上可能达到的最佳实现是什么。另外，如果成功了，来告诉助教，我们会很感兴趣的。;-)

注意：过去有些学生在这个问题上卡住了（想得太复杂）。我们期望一个简单的答案，但运行这个问题得到的结果可能会在你脑海中引发更多问题。欢迎来和助教交流。
